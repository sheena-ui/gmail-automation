Privacy Policy for Inbox Extractor

Last Updated: May 6th, 2025

This Privacy Policy explains how the application Inbox Extractor handles your information when using Google services.

App Overview

Inbox Extractor is a personal tool that uses the Gmail API to access email messages in the developer’s own Gmail account. The app is intended to automate the process of extracting verification codes and links from incoming emails for personal use.

What Google user data is accessed?

The app requests access to Gmail data via the Gmail API in read-only mode. Specifically, it accesses:

  • Email metadata (e.g., sender, subject, timestamps)
  • The message body text of emails

Why is this data accessed?

The app reads Gmail messages to extract verification codes or links used in multi-factor authentication (MFA) or login flows. This is done to support automation tasks during development and testing.

Is any data stored or shared?

No. The application does not store, log, or transmit any Gmail content to third parties. All processing is done locally in memory, and no data is written to disk or sent externally.

Who has access to the data?

Only the authorized user (the developer) has access to Gmail content through this application. The app does not support multiple users or accounts.

Security Measures

The application uses Google's secure OAuth 2.0 authorization flow. The user explicitly grants consent to Gmail read-only access, and the app only uses the access token provided by Google to perform read operations.
